Events

Events are actions or occurrences that happen in the system you are programming, which the system tells you about so you can respond to them in some way if desired.

Eg: if the user selects a button on a webpage, you might want to respond to that action by displaying an information box.

Event Flow

Event flow is the order in which event is received on the web page.

Event Capturing

Event Capturing is the event starts from top element to target element.

Modern browser doesn’t support event capturing by default but we can achieve that by code in JavaScript.

codepen
Event Bubbling

Event Bubbling is the event starts from the deepest element or target element to its parents, then all its ancestors which are on the way to bottom to top.

All the modern browsers have event bubbling as the default way of event flow.

codepen
target vs currentTarget

event.target – is the “target” element that initiated the event, it doesn’t change through the bubbling process.

event.currentTarget - is the “current” element, the one that has a currently running handler on it.

Event Phase

The eventPhase read-only property of the Event interface indicates which phase of the event flow is currently being evaluated.

Constant Value Description
Event.NONE 0 No event is triggered.
Event.CAPTURING_PHASE 1 The event flow is in capturing phase.
Event.AT_TARGET 2 The event flow is in target phase.
Event.BUBBLING_PHASE 3 The event flow is in bubbling phase.
